# prd_云测试服务01.md

## 最简单信息摘要（极简版）

- 需求目标：在既有「开发 > 云测试服务」leaf 上迭代主页面，支撑云测试任务管理与模式管理。
- 页面数量：本期 1 页（仅主页面）。
- 核心入口与跳转：侧栏进入主页面；报告页与对比能力均延期至二期，不在本期实现。
- 模版组合：`LayoutSpec.md`（壳层） + `InfoQuery.md`（主页面列表/筛选结构）。
- 默认状态：页面默认展示 mock 数据（非空态）；同时保留空态规范用于后续实现。
- 风险/待确认项：无阻塞项；二期需补充报告页/对比页信息架构与模版选型。

## 3. 页面模版选型

| 页面 | 主模版 | 变体 | 组合模版 | 选型依据 |
|---|---|---|---|---|
| 云测试服务（主页面） | `InfoQuery.md` | 单页多层 Tab 列表管理（一级 Tab + 二级 Tab） | `LayoutSpec.md` + `InfoQuery.md` | 页面核心是筛选 + 表格 + 操作列，且存在管理类 Tab 切换，和 InfoQuery 模版匹配度最高。 |

### 3.1 结构说明

- 壳层：严格复用 `LayoutSpec`（固定 Header、固定 Sidebar、内容区 32/24/24）。
- 业务区模块顺序：
  1. 页头（标题 + 功能说明文字链）
  2. 常显 Banner
  3. 一级 Tab：测试管理 / 模式管理（默认测试管理）
  4. 业务区（筛选区 + 表格区）
- 模式管理内包含二级 Tab：AI 自动玩 / 脚本定制。

## 4. 页面内元素

## 页面：云测试服务（主页面）

### 4.1 顶部与提示区

| 模块 | 元素/字段 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| 页头 | 页面标题「云测试服务」 | 文本 | 是 | 位于业务内容顶部左侧 |

### 4.2 一级 Tab：测试管理（默认）

#### 4.2.1 筛选区

| 模块 | 元素/字段 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| 筛选区 | 测试ID | 搜索输入 | 否 | 支持关键字检索 |
| 筛选区 | 测试版本 | 搜索输入 | 否 | 支持版本检索 |
| 筛选区 | 提交人 | 搜索输入 | 否 | 支持提交人检索 |
| 筛选区 | 测试时间 | 日期区间选择 | 否 | 精确到日期 |
| 筛选区 | 测试模式 | 下拉选择 | 否 | 选项：AI 自动玩、脚本定制 |
| 筛选区 | 测试状态 | 下拉选择 | 否 | 选项：排队中、测试中、已完成、已取消、服务异常 |
| 筛选区 | 新建测试 | 主按钮 | 是 | 触发新建测试流程（本期可先占位） |

#### 4.2.2 表格区

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| 测试ID | 文本（10字符） | 是 | 支持截断展示 |
| 测试版本 | 文本 | 是 | 版本号/分支信息 |
| 测试模式 | 枚举 | 是 | AI 自动玩 / 脚本定制 |
| 测试状态 | 状态标签 | 是 | 排队中、测试中、已完成、已取消、服务异常 |
| 启动性能 | 百分制评分（可排序） | 是 | 数值排序 |
| 运行性能 | 百分制评分（可排序） | 是 | 数值排序 |
| 网络性能 | 百分制评分（可排序） | 是 | 数值排序 |
| 兼容性 | 百分制评分（可排序） | 是 | 数值排序 |
| 提交人 | 文本 | 是 | 随机人名 mock |
| 提交时间 | 时间 | 是 | 精确到分钟 |
| 备注 | 文本 | 否 | 可为空 |
| 操作（右侧固定） | 文字按钮组 | 是 | 根据状态展示差异操作 |

#### 4.2.3 操作规则

- 非「已取消 / 服务异常」：显示「报告」「对比」。
- 「已取消 / 服务异常」：显示「重新测试」。
- 状态 hover：
  - 排队中、测试中：展示预计时间
  - 服务异常：展示异常原因
